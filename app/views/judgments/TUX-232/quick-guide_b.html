Guide to completing the metadata file

This guide provides a quick overview of how to complete the metadata file effectively. Below, you will find essential information on the metadata you need to fill in, ensuring that all necessary details are accurately captured. To assist you further, we have included a table that serves as a guide to filling in the spreadsheet. By following these instructions and using the table as a reference, you can efficiently complete the metadata file with ease. This is generic text do not consider. test test

Mandatory attributes

These are mandatory attributes in the file and must not be edited.

filepath

  • Details: This is the file path/folder structure extracted upon upload, do not modify.
  • Format: Mandatory

filename

  • Details: This is the file name extracted upon upload, do not modify.
  • Format: Mandatory

date last modified

  • Details: This is the date last modified extracted upon upload, do not modify.
  • Format: Mandatory

Other attributes

date of the record

  • Details: If the date last modified is not meaningful, please provide the date of the record. Optional
  • Format: YYYY-MM-DD (for example: 2020-03-01)

description

  • Details: Provide a description which gives more context to the record. Recommended.
  • Format: Text, max 8000 characters

former reference

  • Details: Provide any reference which may have been used during the life of the record (it must not contain any line breaks). If you have multiple former references, these should be separated by semi-colons.Optional.
  • Format: Text, max 255 characters. (For example: 'CRDA/12')

closure status

  • Details: Closed records must have been approved by the Advisory Council. Only the words 'Open' or 'Closed' will be accepted. This is case sensitive and variations on these words will not be accepted. Mandatory for closed record.
  • Format: Open or Closed, default: Open

closure start date

  • Details: Closed record: Must match date of the record (if provided) else date last modified, must also align to date provided on closure application. Mandatory for closed record.
  • Format: YYYY-MM-DD (For example: '2020-03-01')

closure period

  • Details: Closed record: Provide a list of the number of years closure for each foi exemption code, in corresponding order. Mandatory for closed record.
  • Format: Semi-colon separated list of numbers between 1 and 150 (with no spaces). For example: '40;80'.

foi exemption code

  • Details: Closed record: Provide the FOI exemption codes used to justify closure for the record, as approved by the Advisory Council This should be the same as the code(s) on the closure schedule for the record. Mandatory for closed record.
  • Format: Semi-colon separated list (with no spaces). For example: 27(1);33.

foi schedule date

  • Details: Closed record: This date must match the one found on the closure schedule. It can be found on the bottom right hand corner of each page. Mandatory for closed record.
  • Format: YYYY-MM-DD (for example: 2020-03-01)

is filename closed

  • Details: Closed record: Yes means the filename should also be closed. Mandatory for closed record.
  • Format: Yes or No, default: No

alternate filename

  • Details: Closed record: If the filename of this record is closed, please provide an alternate filename (without the file extension) to be displayed on Discovery while the record is closed. Mandatory if filename is closed.
  • Format: Text, max 8000 characters (For example: 'filename1')

is description closed

  • Details: Closed record: Yes means the description should also be closed.Mandatory for closed record.
  • Format: Yes or No, default: No.

alternate description

  • Details: Closed record: If the description of this record is closed, please provide an alternate description to be displayed on Discovery while the record is closed.Mandatory if description is closed.
  • Format: Text, max 8000 characters. (For example: 'This is an alternative description')

language

  • Details: Indicate if the record is in English, Welsh or both (English;Welsh). This is case sensitive and variations on these words will not be accepted. If you have records in other languages, please contact your digital transfer advisor. Mandatory
  • Format: Semi-colon separated list (with no spaces) default: English. (For example: 'English;Welsh')

translated filename

  • Details: Provide a translation of the title (without the file extension). Optional.
  • Format: Text, max 8000 characters. (For example: 'filenameEnglish')
